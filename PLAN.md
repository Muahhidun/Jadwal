# План проекта: приложение напоминаний о поклонении

*Рабочее название: Jadwal (обсуждается). Версия плана: 1.0 от 04.07.2026.*

---

## 1. Концепция

**Не «ещё одно приложение времён молитв», а ассистент поклонения.**

Sajde kz, Muslim Pro и подобные отвечают на вопрос «когда намаз?». Наше приложение отвечает на вопрос **«какое поклонение я могу совершить прямо сейчас и что я рискую пропустить?»** — и напоминает о временных окнах, у которых есть основание в Сунне:

| Напоминание | Окно | Действие по тапу |
|---|---|---|
| Утренние зикры | После Фаджра, до восхода | Открывается сборник утренних зикров |
| Вечерние зикры | После Асра, до Магриба | Открывается сборник вечерних зикров |
| Сура аль-Кахф | Пятница, за 1–2 часа до Джума (настраивается) | Текст суры / напоминание прочитать |
| Дуа в пятницу | Пятница, за час до Магриба | Экран с напоминанием о часе принятия дуа |

Плюс пользовательские напоминания: привязанные к любой молитве («за X минут до / через X минут после») или к фиксированному времени.

**Слоган-позиционирование:** «Не пропусти награду» / «Расписание твоего поклонения».

## 2. Ключевые решения (зафиксированы)

1. **География:** Казахстан-first. Потом СНГ и дальше.
2. **Позиционирование:** «рядом с Sajde», не вместо. Главный экран — про поклонение и зикры. Времена молитв есть внутри (без них напоминаниям не поверят), но как справочный блок, не как главный экран.
3. **Времена молитв:** данные ДУМК (муфтият, API namaz.muftyat.kz) для Казахстана — совпадаем с Sajde минута в минуту. Астрономический расчёт (библиотека adhan) — офлайн-запас и для пользователей за пределами КЗ.
4. **Монетизация:** бесплатно, ради Аллаха. Без рекламы. Возможна кнопка «поддержать проект» позже.
5. **Языки:** русский, казахский; арабский для текстов зикров. Структура карточки зикра: арабский → транслитерация → перевод → достоинство/польза (+ тафсир, если есть). Каждый слой включается/отключается в настройках.
6. **⚠️ Никаких счётчиков-тасбихов.** Принципиальное религиозное ограничение: цифровой счётчик зикра — нововведение (как и чётки), зикр считается на пальцах. Вместо счётчика — отметка «выполнено» (галочка) на каждый зикр: «сказал 33 раза → галочка».
7. **История и постоянство:** храним историю выполнения (календарь), мотивируем постоянство («самое любимое деяние перед Аллахом — постоянное»), но **без геймификации-самообмана**: никаких очков, бейджей, рейтингов, соревнований. Только честная личная картина.
8. **Контент:** собирает и проверяет владелец проекта из достоверных сборников (Хиснуль-Муслим и др.). У каждого зикра — источник (хадис, сборник, степень достоверности).
9. **Команда:** владелец (продукт + контент) + Claude Code (весь код, дизайн, архитектура).
10. **Стек:** Flutter — один код на iOS и Android. Без бэкенда и аккаунтов: все данные локально на устройстве.

## 3. Состав MVP

> ⚠️ **Дизайн финализирован (04.07.2026).** Источник правды по UI — `design_handoff_jadwal/README.md` + `Jadwal Prototype.dc.html` (hi-fi прототип из Claude Design: все экраны, локали ru/kz, тёмная/светлая темы, дизайн-токены, спецификация анимаций, карта навигации). Ключевые решения дизайна, заменяющие ранние наброски ниже: главный экран показывает **одно актуальное дело**, а не дашборд; нижней таб-навигации **нет** — приложение трёхслойное (Главный ⇅ «День и тетрадь» свайпом вверх, Чтение зикров — модальный слой «бумага»); после завершения зикров — сдержанный экран празднования со случайным хадисом; история — «тетрадь постоянства» (зелёный/золотой ромб/красный, без цифр и стриков); темы: тёмная (по умолчанию)/светлая/системная; онбординг 5 шагов, казахский первым. При расхождении этого раздела с хендоффом приоритет у хендоффа. Пресет «оповещать о каждом намазе» (с настройкой по каждому намазу) в прототип не вошёл — добавить в том же стиле в конструктор напоминаний.

### 3.1 Онбординг (первый запуск)
- Выбор языка (рус/каз).
- Запрос геолокации → определение города → **или ручной выбор города** (обязательно: многие не дают геолокацию, и это должно работать).
- Запрос разрешения на уведомления с объяснением зачем («мы напомним об утренних и вечерних зикрах…»). На Android 13+ и iOS это отдельные системные запросы.
- Показ базовых напоминаний с переключателями — пользователь сразу видит ценность.

### 3.2 Главный экран «Сегодня»
Покрывает и базовую потребность «когда намаз?», и нашу суть — чтобы приложение могло быть у пользователя единственным:
- **Верхний блок (герой):** ближайшая молитва с обратным отсчётом («Магриб через 42 минуты») + компактная строка всех 6 времён на сегодня.
- **Логика счётчика после азана:** когда время намаза наступило, первые 10–20 минут (настраивается) показываем «после азана прошло X мин» — удобно тем, кто идёт в мечеть (джамаат начинается через 10–20 минут после азана). Только потом автоматически переключаемся на отсчёт до следующего намаза. Это поведение по умолчанию; тап по счётчику переключает режим вручную.
- **Тап по городу** → выбор города вручную. **Тап по дате** → переключение хиджра ⇄ григорианский.
- **Текущее окно поклонения:** «сейчас время вечерних зикров — до Магриба» с кнопкой перехода к зикрам.
- **Чек-лист дня** с галочками: утренние зикры ✓, вечерние зикры, (в пятницу: Кахф, дуа).

Уведомления о самих намазах отдельно не строим: движок напоминаний уже умеет «за 0 минут до молитвы» — включаемый пресет «напоминать о каждом намазе» закрывает базовую потребность без азан-функционала.

### 3.3 Экран зикров (ядро дифференциации)
- Открывается по тапу на уведомление (deep link) и с главного экрана.
- **Постраничный режим:** один зикр на экран, свайп/стрелки для листания; «выполнено» → автопереход к следующему зикру.
- Слои карточки (арабский, транслит, перевод, достоинство+источник) — сворачиваемые заголовки. **Состояние слоя глобальное:** свернул «Транслит» в одном зикре — свернулся во всех и запомнился (это и есть настройка отображения, прямо в чтении).
- Под каждым зикром: количество повторений («33 раза») и кнопка «выполнено». Прогресс сессии = сколько зикров отмечено. Никаких счётчиков нажатий.
- Настройка размера арабского шрифта.

### 3.4 Напоминания
- Список напоминаний — не тумблеры в строке, а **кнопки**: тап по напоминанию открывает его экран настроек (включено; «когда напоминать» — сдвиг относительно молитвы; повторное напоминание, если не отмечено выполненным).
- Базовый набор (4 шт.) с настройкой сдвига у каждого.
- Пресет «оповещать о каждом намазе»: внутри — настройка по каждому намазу отдельно (например, Фаджр — вовремя, Зухр — за 5 мин, Магриб — за 10 мин).
- Свои напоминания: название + привязка («за X мин до / через X мин после» любой из 6 точек: Фаджр, Восход, Зухр, Аср, Магриб, Иша — или фиксированное время) + дни недели.
- Уведомление ведёт на соответствующий контент (зикры) или просто напоминает.

### 3.5 Времена молитв
- Экран на сегодня + на месяц, город, дата по хиджре.
- Без азана/звуков в MVP (это территория Sajde, вернёмся позже, если попросят).

### 3.6 История / постоянство
- Календарь: какие дела выполнены в какие дни.
- Простая сводка («в этом месяце: утренние зикры 18 из 22 дней») — без бейджей и стриков-манипуляций.

### 3.7 Настройки
- Город/геолокация, язык, слои карточки зикра, размер шрифта, управление напоминаниями.
- **Темы оформления:** 3–4 цветовых стиля (базовый — «Изумруд»: глубокий зелёный + золото), светлая/тёмная версия каждого, авто-тема по времени суток. Следить за контрастом тёмных элементов на тёмно-зелёном (замечание владельца по v1 прототипа).
- **Дизайн-направление:** современный консюмерский стиль, а не «сайт с дашбордом» (референсы владельца: Dodo, Drinkit и приложения техно-компаний). 04.07.2026 показаны 4 варианта главного экрана (А свежий минимализм / Б плейфул-карточки / В премиум-ночь / Г iOS-минимализм) — ждём выбора владельца.

### Не входит в MVP (осознанно)
Азан-звуки, кибла, Коран целиком, аудио-озвучка зикров, виджеты, Apple Watch, тёмная тема (или войдёт дёшево — Flutter даёт почти бесплатно), синхронизация между устройствами, аккаунты.

## 4. Технический скелет

### 4.1 Стек
- **Flutter (Dart)** — iOS + Android из одной кодовой базы.
- Локальная БД: **Drift (SQLite)** — зикры, напоминания, история.
- Уведомления: `flutter_local_notifications` (локальные, без пуш-сервера).
- Времена молитв: пакет **adhan_dart** (расчёт) + HTTP-клиент к API муфтията с кешем.
- Геолокация: `geolocator` (разовое, грубое определение) + офлайн-справочник городов КЗ.
- Без сервера. Приватность — конкурентное преимущество: «мы не собираем ваши данные».

### 4.2 Архитектура уведомлений (главный технический риск)
Проблема: время молитв меняется ежедневно → время напоминаний меняется ежедневно, а iOS ограничивает очередь 64 локальными уведомлениями и убивает фоновые процессы.

Решение:
1. При каждом открытии приложения пересчитываем времена и **планируем уведомления на 10–14 дней вперёд** (≈2–4 уведомления в день → укладываемся в лимит 64 с запасом).
2. Фоновое обновление очереди: `WorkManager` (Android) и `BGAppRefreshTask` (iOS) — раз в сутки доп планируют очередь. iOS не гарантирует запуск фоновой задачи, поэтому пункт 1 — основной, пункт 2 — подстраховка.
3. Если пользователь не открывал приложение ~10 дней и фон не сработал — последнее запланированное уведомление мягко зовёт открыть приложение.
4. Смена города/геолокации → полная перепланировка очереди.
5. Годовые таблицы времён для городов КЗ кешируются (и/или зашиваются в приложение) → всё работает офлайн.

### 4.3 Модель данных (эскиз)
- `ZikrCollection` (утренние, вечерние, …) → `Zikr` (араб., транслит, переводы ru/kk, достоинство, источник, повторения, порядок).
- `Reminder` (тип: базовый/свой; привязка: молитва+сдвиг или фикс. время; дни недели; вкл/выкл; действие по тапу).
- `CompletionLog` (дата, что выполнено) — история.
- `PrayerTimesCache` (город, дата, 6 времён, источник: муфтият/расчёт).

### 4.4 Контент-пайплайн
Владелец собирает зикры в структурированную таблицу (Google Sheets/Excel по шаблону: арабский | транслит | перевод ru | перевод kk | достоинство | источник | повторения | сборник). Скрипт конвертирует в JSON, который зашивается в приложение. Обновление контента = обновление приложения (без сервера).

## 5. Конкурентное преимущество (против Sajde kz)

1. **Категория, а не функция.** Мы не «время молитв + функции», мы — «помощник постоянства в поклонении». Sajde говорит когда молиться; мы помогаем не пропустить зикры, Кахф и час дуа. Пользуются обоими — и это нормально для старта.
2. **Опыт «уведомление → сразу зикры».** Ни у кого нет цельного сценария: напоминание в правильное окно → тап → готовый сборник с отметками. У конкурентов зикры — мёртвый раздел меню.
3. **Достоверность как бренд.** Источник у каждого зикра, никаких слабых хадисов без пометки, никаких счётчиков-нововведений. Для практикующей аудитории, которая разборчива в вопросах Сунны, это громкий сигнал — и это аудитория ранних последователей, которая приводит остальных.
4. **Бесплатно, без рекламы, без сбора данных.** Садака-джария. Сильный аргумент в описании стора и в устах пользователей.
5. **Канал привлечения:** казахстанские исламские блогеры/устазы, Telegram-каналы, мечети. Продукт «расскажи брату» — виральность заложена в сути (напомнить другому о зикре — тоже награда).

## 6. Дорожная карта

**Этап 0 — Скелет и дизайн (мы здесь).**
План ✓ → структура контента → интерактивный прототип экранов (согласуем внешний вид до кода) → варианты названия и проверка доступности в сторах.

**Этап 1 — MVP-разработка.**
Flutter-проект: времена молитв (муфтият + adhan) → движок напоминаний → экран зикров → главный экран → история → настройки. Параллельно: владелец собирает контент по шаблону; регистрация Google Play ($25) и Apple Developer ($99/год — оформить заранее, проверка занимает дни).

**Этап 2 — Закрытая бета.**
Google Play Internal Testing + TestFlight. 20–50 знакомых практикующих. Главная метрика беты: **доставляются ли уведомления вовремя на реальных устройствах** (особенно на китайских Android — Xiaomi/Huawei агрессивно убивают фон, нужна инструкция «как разрешить»).

**Этап 3 — Релиз в Казахстане.**
Публикация, страницы сторов (ru/kk), посев через блогеров/каналы, сбор отзывов.

**Этап 4 — Развитие (по обратной связи).**
Кандидаты: виджеты на главный экран, аудио, полные времена молитв с азаном (если попросят «заменить Sajde»), другие сборники дуа (перед сном, после намаза), английский язык и выход за КЗ.
