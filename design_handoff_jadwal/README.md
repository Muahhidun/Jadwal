# Handoff: Jadwal — ассистент поклонения (MVP)

## Overview
Jadwal — мобильное приложение-«ассистент поклонения» для практикующих мусульман (старт — Казахстан, аудитория 18–40). Приложение знает времена молитв для города пользователя, напоминает об окнах поклонения (утренние/вечерние зикры, сура аль-Кахф, час дуа в пятницу), открывает тексты зикров (арабский, транслитерация, перевод, достоинство, источник), позволяет отметить выполненное и видеть картину постоянства по дням.

**Жёсткие продуктовые рамки (нарушать нельзя):**
- Никаких счётчиков-тасбихов и подсчёта нажатий — зикр отмечается выполненным целиком.
- Никакой геймификации: без очков, бейджей, стриков-огоньков, соревнований.
- У каждого религиозного текста виден источник.
- Уважительный, спокойный тон. Бесплатно, без рекламы.

## About the Design Files
Файлы в этом пакете — **дизайн-референсы, созданные в HTML** (прототипы, показывающие задуманный вид и поведение), а не production-код для прямого копирования. Задача — **воссоздать эти дизайны в целевом стеке** (React Native / Flutter / SwiftUI+Compose — на выбор команды; окружения ещё нет, выбери наиболее подходящий фреймворк для кроссплатформенного мобильного приложения) с использованием его паттернов.

`Jadwal Prototype.dc.html` — основной интерактивный прототип. Логика — в `<script>`-классе внутри файла (plain JS, React-подобный класс: `state`, `renderVals()` возвращает данные и обработчики для шаблона). Читай шаблон (разметку между `<x-dc>`) + класс как единый источник правды по вёрстке, строкам и переходам.

`Jadwal Concepts.dc.html` — ранние концепты (для истории решений, реализовывать не нужно).

**High-fidelity.** Цвета, типографика, отступы, копирайтинг и анимации — финальное направление. Воссоздавать точно.

## Navigation Map
```
Онбординг (один раз): Язык → Город → Уведомления → Тема → Постер → Главный
Главный ──«Читать зикры» / тап по задаче «Вечерние»──▶ Чтение (модально, поверх)
Чтение ──✕──▶ Главный (без отметки)
Чтение ──«Завершить ✓» на последнем──▶ Празднование ──«Вернуться»──▶ Главный (уже в след. состоянии)
Главный ──свайп вверх / тап по хэндлу──▶ День ──свайп вниз / тап по хэндлу──▶ Главный
День ──«Напоминания»──▶ Настройки ──←──▶ День
Настройки ──«+ Добавить»──▶ Своё напоминание ──«Сохранить» или ←──▶ Настройки
Кнопки «Тема» и «Тіл · Язык» на «Дне» переключают на месте, без перехода.
Пуш-уведомление (в продукте) ──тап──▶ сразу Чтение соответствующего окна.
```
Нижней таб-навигации нет намеренно — приложение трёхслойное: главный / день (вертикальный жест) / чтение (модальный слой).

## Animation Spec (точные значения)
- **rise** (все обычные переходы экранов): `opacity 0→1, translateY(16px)→0`, 350ms ease, на входящем экране.
- **dayup** (открытие «Дня»): `translateY(100%)→0` + `blur(10px)→blur(4px) на 60%→0`, 450ms `cubic-bezier(.22,.8,.3,1)`. Закрытие — обратный слайд. В проде: interruptible pan-жест; таймер — shared element с главного экрана.
- **hintbounce** (хэндл свайпа на главном): цикл 5s infinite, delay 2s; кадры: 0/72/100% — покой, 78% `translateY(-8px)`, 84% — 0, 90% `translateY(-4px)`, 96% — 0. Ease-in-out.
- **pop** (галочка/бейдж): `scale .6→1.08 (70%)→1`, 500–550ms ease, с fade-in.
- **confetti** (празднование): ~16 частиц 5–9px, из точки над галочкой, разлёт радиально на 90–190px (dy смещён вверх на −60px), `rotate 0→560deg`, `opacity 1→0`, длительность 0.9–1.4s, delay 0.35–0.56s, `cubic-bezier(.2,.7,.4,1)`.
- **flydown** (галочка-призрак): копия галочки поверх основной; 0–55% на месте, затем `translateY(+236px) scale(.22)` + fade, 1.6s `cubic-bezier(.4,0,.6,1)`, delay 0.9s. Конечная точка = сегодняшняя ячейка недельной полоски.
- **landpop** (ячейка «сегодня» в полоске): невидима до 82% таймлайна 3s, затем `scale .5→1.25 (92%)→1` + появление. Итог: галочка «долетела» и встала.
- **Тумблеры**: фон и knob — transition 200ms.
- Переключение темы/языка — мгновенное (без анимации).

## Interactions & Behavior
- Переходы экранов: см. Animation Spec выше.
- Все тап-таргеты ≥44px.
- Язык: полная локализация ru/kz (все строки в прототипе, объект `strings(lang)` в классе — использовать как i18n-словарь). Казахский всегда первым в двуязычных подписях.
- Темы: тёмная (default) / светлая / системная. Экран чтения и празднования всегда «бумага».
- Напоминания-пуши: по открытию окна поклонения; тап на пуш → сразу экран чтения.
- Демо-время в прототипе зафиксировано (пятница 20:11, Алматы). В продукте — реальный расчёт времён молитв по городу (метод расчёта — ДУМК по умолчанию), таймеры живые.

## State Management
- `lang` ('ru'|'kz'), `theme` ('dark'|'light'|'system'), `city`
- Онбординг пройден (bool, показывается один раз)
- Дневные отметки: morning/kahf/evening/dua (+ timestamp выполнения)
- История по дням для тетради: full / partial / missed / future
- Тумблеры напоминаний + массив пользовательских напоминаний {name, bind, offset, days, on}
- Текущее окно вычисляется из времён молитв: главный экран показывает первое невыполненное открытое окно, иначе таймер до следующей молитвы
